# 数据服务配置

数据服务（`data_services`）提供了内置的数据填充和展示功能。

## 配置概览

数据服务按需加载，页面使用时才会加载对应的 JS 文件。

## 常用服务

### siteinfo - 站点信息

自动提取网页标题和图标。

```yaml
data_services:
  siteinfo:
    js: /js/services/siteinfo.js
    api: https://api.example.com/site_info/v1?url={href}
```

### ghinfo - GitHub 信息

显示 GitHub 仓库信息。

```yaml
data_services:
  ghinfo:
    js: /js/services/ghinfo.js
```

**使用方法**：

```markdown
{% ghinfo username/repo %}
```

### friends - 友链

展示友情链接。

```yaml
data_services:
  friends:
    js: /js/services/friends.js
```

### timeline - 时间线

展示时间线内容。

```yaml
data_services:
  timeline:
    js: /js/services/timeline.js
```

## 贡献者服务

### contributors - 贡献者列表

```yaml
data_services:
  contributors:
    edit_this_page:
      '_posts/': https://github.com/username/repo/blob/main/source/_posts/
    js: /js/services/contributors.js
```

## 完整配置示例

```yaml
data_services:
  siteinfo:
    js: /js/services/siteinfo.js
  ghinfo:
    js: /js/services/ghinfo.js
  friends:
    js: /js/services/friends.js
  timeline:
    js: /js/services/timeline.js
  giscus:
    js: /js/services/giscus_latest_comment.js
  contributors:
    edit_this_page:
      '_posts/': https://github.com/username/repo/blob/main/source/_posts/
    js: /js/services/contributors.js
```

## 最佳实践

1. 仅启用需要的服务
2. 配置正确的 API 地址
3. 测试服务是否正常工作
