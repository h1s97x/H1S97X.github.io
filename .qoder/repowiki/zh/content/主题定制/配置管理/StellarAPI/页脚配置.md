# 页脚配置

页脚配置（`footer`）包含社交链接、站点地图和版权信息。

## 配置结构

```yaml
footer:
  social: {}
  sitemap: []
  content: ''
```

## social - 社交链接

### 配置示例

```yaml
footer:
  social:
    github:
      icon: '<img src="https://example.com/github.svg"/>'
      url: https://github.com/username
    twitter:
      icon: '<img src="https://example.com/twitter.svg"/>'
      url: https://twitter.com/username
```

## sitemap - 站点地图

### 配置示例

```yaml
footer:
  sitemap:
    - title: 博客
      items:
        - '[近期发布](/)'
        - '[分类](/categories/)'
        - '[标签](/tags/)'
    - title: 页面
      items:
        - '[关于](/about/)'
        - '[联系](/contact/)'
```

## content - 版权信息

### 配置示例

```yaml
footer:
  content: |
    本站由 [作者名](/) 使用 [Stellar]({theme.tree}) 主题创建。
    本博客所有文章除特别声明外，均采用 [CC BY-NC-SA 4.0](https://creativecommons.org/licenses/by-nc-sa/4.0/) 许可协议。
```

## 完整配置示例

```yaml
footer:
  social:
    github:
      icon: '<img src="/images/github.svg"/>'
      url: https://github.com/username
  sitemap:
    - title: 博客
      items:
        - '[首页](/)'
        - '[归档](/archives/)'
    - title: 其他
      items:
        - '[RSS](/atom.xml)'
        - '[站点地图](/sitemap.xml)'
  content: |
    © 2025 作者名 | [CC BY-NC-SA 4.0](https://creativecommons.org/licenses/by-nc-sa/4.0/)
```

## 最佳实践

1. 社交链接不宜过多（3-5 个）
2. 站点地图分组清晰
3. 版权信息简洁明了
