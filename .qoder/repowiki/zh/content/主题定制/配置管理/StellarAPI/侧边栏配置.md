# 侧边栏配置

侧边栏是 Stellar 主题的核心导航区域，包含 Logo 区域和主功能导航菜单。

## logo - Logo 区域配置

Logo 区域位于侧边栏左上角，包含头像、标题和副标题。

### 配置项

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| avatar | string | - | 头像图片链接，支持 Markdown 链接语法 |
| title | string | - | 站点标题，支持 HTML 标签 |
| subtitle | string | - | 副标题，支持切换效果 |

### 配置示例

#### 基础配置

```yaml
logo:
  avatar: '[{config.avatar}](/about/)'
  title: '[{config.title}](/)' 
  subtitle: '{config.subtitle}'
```

#### 使用自定义图片

```yaml
logo:
  avatar: '[https://example.com/avatar.png](/about/)'
  title: '[我的博客](/)' 
  subtitle: '记录技术与生活'
```

#### 使用 HTML 标签

```yaml
logo:
  title: '[<img no-lazy height="32px" src="/images/logo.svg"/>](/)'
  subtitle: '文字1 | 文字2'  # 鼠标悬停时切换到"文字2"
```

### 配置说明

#### avatar 字段
- 支持 Markdown 链接语法：`[图片URL](跳转链接)`
- 可以使用 `{config.avatar}` 引用站点配置的头像
- 图片会自动裁剪为圆形

#### title 字段
- 支持 Markdown 链接语法
- 支持 HTML 标签（如 `<img>`）
- 可以使用 `{config.title}` 引用站点配置的标题

#### subtitle 字段
- 支持纯文本
- 使用 `|` 分隔两段文字，实现鼠标悬停切换效果
- 可以使用 `{config.subtitle}` 引用站点配置的副标题

## menubar - 主功能导航菜单

主功能导航菜单位于侧边栏中部，提供快速访问主要页面的入口。

### 配置项

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| columns | number | 4 | 每行显示的菜单项数量 |
| items | array | - | 菜单项列表 |

### items 菜单项配置

每个菜单项包含以下字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 菜单项唯一标识符 |
| theme | string | 否 | 主题色（HEX 格式，如 `#1BCDFC`） |
| icon | string | 是 | 图标（支持 Iconify） |
| title | string | 是 | 显示标题 |
| url | string | 是 | 跳转链接 |

### 配置示例

#### 标准配置

```yaml
menubar:
  columns: 4
  items:
    - id: post
      theme: '#1BCDFC'
      icon: solar:documents-bold-duotone
      title: 博客
      url: /blog/
    - id: wiki
      theme: '#3DC550'
      icon: solar:notebook-bold-duotone
      title: 文档
      url: /wiki/
    - id: notes
      theme: '#FF9800'
      icon: solar:pen-new-square-bold-duotone
      title: 笔记
      url: /notes/
    - id: about
      theme: '#F44336'
      icon: solar:user-bold-duotone
      title: 关于
      url: /about/
```

#### 三列布局

```yaml
menubar:
  columns: 3
  items:
    - id: post
      icon: solar:documents-bold-duotone
      title: 博客
      url: /
    - id: wiki
      icon: solar:notebook-bold-duotone
      title: 文档
      url: /wiki/
    - id: about
      icon: solar:user-bold-duotone
      title: 关于
      url: /about/
```

### 图标选择

推荐使用 [Iconify](https://icon-sets.iconify.design/) 图标库：

**常用图标集**：
- `solar:` - Solar Icons（推荐，风格统一）

## 最佳实践

### 1. 菜单项数量

- **推荐 4-6 个**：保持简洁，避免过多选项
- 超过 6 个时考虑使用下拉菜单或分组

### 2. 布局选择

- **4 列**：适合 4、8 个菜单项
- **3 列**：适合 3、6、9 个菜单项
- **2 列**：适合移动端或极简风格

### 3. 图标风格

- 保持同一图标集，确保视觉统一
- 推荐使用 `solar:` 系列的 `bold-duotone` 风格
- 避免混用不同风格的图标

### 4. 主题色搭配

- 使用对比色区分不同类型的内容
- 保持整体色调协调
- 考虑色盲用户，避免仅用颜色区分

### 5. URL 规范

- 使用绝对路径（以 `/` 开头）
- 目录 URL 以 `/` 结尾
- 保持 URL 结构清晰

## 常见问题

### 菜单项不显示？

1. 检查 `id` 是否唯一
2. 确认 `url` 对应的页面存在
3. 验证 YAML 缩进是否正确

### 图标不显示？

1. 检查图标名称是否正确
2. 确认网络可以访问 Iconify CDN
3. 尝试使用其他图标集

### 如何添加外部链接？

```yaml
items:
  - id: github
    icon: mdi:github
    title: GitHub
    url: https://github.com/username
```

### 如何高亮当前页面？

菜单项的 `id` 会与页面的 `menu_id` 匹配，匹配时自动高亮。在页面 front-matter 中设置：

```yaml
---
menu_id: post
---
```
